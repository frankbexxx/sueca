==============================
Rules:
==============================

Jogadores: 4 jogadores divididos em 2 duplas

Baralho: 40 cartas (baralho limpo, sem 8, 9, 10 e curingas)

Objetivo: Fazer 60 pontos ou mais

Distribuição:
Cada jogador recebe 10 cartas
A última carta distribuída determina o naipe de trunfo

Hierarquia das cartas (do mais fraco ao mais forte): 2, 3, 4, 5, 6, Q, J, K, 7, A

Pontuação das cartas:
Q (Dama): 2 pontos
J (Valete): 3 pontos
K (Rei): 4 pontos
7: 10 pontos
A (Ás): 11 pontos
Outras cartas: 0 pontos

Regras de jogo:
O distribuidor joga primeiro
Obrigatório seguir o naipe da primeira carta jogada
Se não tiver o naipe, pode jogar qualquer carta ou trunfo
Trunfo vence qualquer outra carta
Vence a rodada a carta mais alta do naipe jogado ou o trunfo mais alto

Blefe:
Pode-se contestar se suspeitar de blefe
Se o blefe for confirmado, a dupla que contestou ganha a rodada
Se não houver blefe, a dupla acusada ganha a rodada

Pontuação do jogo:
Ganha a dupla que fizer 61 pontos ou mais
Empate em 60 pontos: próxima rodada vale 2 pontos
91 pontos ou mais: vale 2 vitórias
Ganhar todas as rodadas: 4 vitórias

Regra de desempate:
Em caso de empate (ambas as duplas fazem 60 pontos), a próxima rodada vale 2 pontos.

Vitória:
A primeira dupla a completar 4 pontos (vitórias) ganha o jogo
Estas são as regras básicas para jogar Sueca conforme o texto fornecido.

Model-View-Controller pattern
==============================

directory structure

sueca/
│
├── models/
│   ├── __init__.py
│   ├── card.py
│   ├── deck.py
│   ├── player.py
│   └── game.py
│
├── views/
│   ├── __init__.py
│   ├── console_view.py
│   └── gui_view.py
│
├── controllers/
│   ├── __init__.py
│   └── game_controller.py
│
├── utils/
│   ├── __init__.py
│   └── constants.py
│
├── tests/
│   ├── __init__.py
│   ├── test_models.py
│   ├── test_views.py
│   └── test_controllers.py
│
├── main.py
└── README.md

explanation:
Here's a brief explanation of each component:

1. Models: Represent the data and business logic
card.py: Defines the Card class
deck.py: Defines the Deck class
player.py: Defines the Player class
game.py: Defines the Game class with core game logic

2. Views: Handle the presentation layer
console_view.py: For text-based interface
gui_view.py: For potential future graphical interface

3. Controllers: Manage the flow between models and views
game_controller.py: Coordinates game flow and user interactions

4. Utils: Contains utility functions and constants
constants.py: Stores game constants like card values, suits, etc.

5. Tests: Contains unit tests for each component

6. main.py: Entry point of the application
